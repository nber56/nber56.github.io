了解学习Wwise SDK

即使在做过音乐音效，参与游戏开发之后还是有着生疏感，如果是一款大型游戏，如果是其他游戏引擎，如果是创意形式的音频需求，该怎么办呢？对于刚入门的新人，是非常有挑战的课题。所以学习wwise更加底层的内容是很有帮助的。附上溪夜的文章地址
【WAAPI适合所有人]（https://blog.audiokinetic.com/zh/everyone-can-use-waapi-overview/）
内容很有启发.但是深入这部分也有一定的前置要求。具体看原文便可理解。
结合我个人的情况，我也只能是很粗浅的学习一下。我学过c,python以及其他一些语言，实际能力非常浅。下面我具体分析一下学习的内容和方向。

首先，查看wwise用户文档，wwise SDk区域，有不同的章节，可以按照顺序也可以选择阅读。大致三部分

第一部分

开始前的准备工作/声音引擎集成纵览/示例/进阶内容/参考材料
关于SDK原理，结构的描述，还有关于具体声音实例，具体应用实例。这一部分需要不断理解。
其中
(Wwise SDK 2022.1.6 > 声音引擎集成纵览 > 将 Wwise 元素集成到游戏中)
介绍了wwise工程中各模块进入的解释：举例如下
1 RTPC的函数原理，需要在不同使用状况下正确配置，避免游戏内混乱
2 event在游戏内会出现发送事件和实际播放声音的延迟，具体的延迟计算

第二部分

版本和平台信息
适配问题是一个重要的方面

第三部分

模块/命名空间/类/文件
这一部分在实际开发和学习中需要不断理解和总结，内容比较多。个人感觉难度稍高
举例：获取有关RTPC的信息
1 查找使用函数ak.wwise.core.object.get
2 选择查询参数，比如用WAQL,引擎内获取GUID填入 

{
"waql": "\"{67A2C1B5-4FBF-4938-A86C-4AEE7E84822A}\""
}

3 选择返回选项，即是查询信息 

{
"return": [
    "type",
    "path",
    "@PropertyName",
    "@ControlInput",
    "@Curve"
]
}
